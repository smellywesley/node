# NODE Five-Minute Proof Demo And Sales Plan

Updated: 2026-07-07
Status: implementation brief for first AppSec-focused proof recording and S$750 pilot sales

## Core Positioning

NODE is not another coding agent. NODE is the control layer around the agents a team is already adopting.

The five-minute proof should sell one outcome:

> Bring one risky coding-agent workflow. NODE will prove what the agent was allowed to do, what was blocked, who approved it, what it cost, and what audit evidence remains.

## Primary Buyer

Optimize the first proof for AppSec and security engineering teams.

Why this buyer first:

- They are being asked to approve AI coding agents.
- They cannot treat prompt instructions as controls.
- They need evidence of denied actions, approval gates, cost records, and audit bundles.
- They can champion a private/local pilot before hosted enterprise procurement is ready.

Secondary buyers are Platform/DevEx leaders, regulated engineering teams, and AI-native startups scaling agent usage.

## Five-Minute Story

1. Market shock: AI coding agents are already producing real software work.
2. Failure risk: prompt instructions are not an enforceable boundary.
3. NODE control map: Request -> Policy Gate -> NODE Control -> Sandbox -> Cost Meter -> Audit Bundle.
4. Live run: backend write is allowed and approval-gated; frontend write is denied.
5. Proof packet: status, events, hashes, cost, and artifacts.
6. Audit bundle: exportable evidence survives after the run.
7. Paid ask: S$750 for one private/local controlled workflow proof.

## What The Pilot Includes

The S$750 private pilot includes:

- one repository-shaped workflow;
- one policy boundary;
- one sandboxed run;
- one approval gate;
- one spend cap;
- one denied action;
- one exported audit bundle;
- one written go/no-go recommendation.

It excludes hosted multi-tenant SaaS, SSO/RBAC, managed model credits, SOC 2 claims, HIPAA readiness, and broad custom agent work.

## Sales Talk Track

Opening:

"AI coding agents are already inside software delivery. The blocker is not whether they can code. The blocker is whether security can prove what happened."

Problem:

"Prompt instructions are not a security boundary. If an agent can write files or call tools, the control must live outside the model."

Proof:

"NODE runs the agent as a controlled process. It enforces file/tool boundaries, pauses risky writes for approval, records cost, and exports an audit bundle."

Ask:

"Bring one risky workflow. If we can turn it into a controlled proof, the S$750 pilot buys the decision artifact your security/platform team needs."

## Target Account Filters

Strong fit:

- already testing Cursor, Claude Code, Codex, Copilot agents, Devin, or MCP toolchains;
- has a repository workflow with a clearly dangerous action;
- needs AppSec, platform, or leadership approval;
- can run local/private Docker and BYOK;
- can pay for a narrow proof before a hosted enterprise rollout.

Weak fit:

- wants a chatbot, not repository controls;
- needs hosted SSO/RBAC before any local proof;
- wants managed model credits included;
- cannot name one risky workflow;
- wants broad consulting instead of one controlled run.

## Acceptance Criteria

The demo and sales packet are ready when:

- the recording is under five minutes;
- the proof shows one denied action, one approval event, one cost record, and one audit export;
- the proof packet includes hashes and repeatable evidence;
- the CTA sells the S$750 private/local pilot;
- all market claims include sources;
- the script does not claim SOC 2, HIPAA, hosted SaaS, tenant isolation, RBAC, or managed billing readiness.
