# Decision: Per-process allowlist egress proxy

- Date: 2026-06-10
- Status: accepted

Workers with no declared network destinations use Docker `network=none`.
Workers with destinations receive a dedicated internal Docker network and a
small proxy sidecar. Only the proxy joins the default bridge. The proxy permits
HTTPS CONNECT to exact declared hostnames on port 443 and rejects private,
loopback, link-local, multicast, unspecified, and reserved addresses.

This keeps network policy outside the probabilistic agent process and prevents
direct socket bypass from the worker. The remaining trust assumptions are the
Docker engine, host DNS, and the proxy implementation.
